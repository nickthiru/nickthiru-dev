# Policies

## Always

- Always select a template from `references/05_TEMPLATES.md` before writing.
- Always identify the track (engineering, business, product) before drafting.
- Always apply SEO optimization to every post.
- Always run the quality checklist before publishing.
- Always apply `voice-and-style` voice policies during the style rewrite step.
- Always include engagement elements (question, CTA, related links).
- Always set up proper frontmatter with all required fields.
- Always target 800–1500 words per post.

## Never

- Never publish without passing the quality checklist.
- Never skip SEO optimization.
- Never write without selecting a template first.
- Never override `voice-and-style` voice policies.
- Never process files from `src/content/todo/` unless the user explicitly mentions specific files.
- Never publish code examples that have not been tested.

## SEO Requirements

- **Meta description**: 150–160 characters, includes keywords naturally.
- **Title tags**: 50–60 characters, compelling and keyword-rich.
- **Headings**: Proper H1 → H2 → H3 hierarchy.
- **Internal links**: Link to 2–3 relevant previous posts.
- **Image optimization**: Alt text, descriptive filenames.
- **URL structure**: Short, descriptive, includes keywords.
- **Mobile-friendly**: Responsive formatting.

### Voice + SEO Balance

- Keywords first, voice second — include keywords naturally in authentic voice.
- Meta descriptions in Nick's voice — write as if talking to a friend.
- Headings as questions — conversational heading questions that include keywords.
- Internal links with context — "Here's how I solved [keyword] problem."

## Technical Content Standards

- Test all code examples before publishing.
- Include imports and necessary dependencies.
- Demonstrate proper error handling.
- Specify relevant library/framework versions.
- Add explanatory comments for complex code.

## Quality Checklist

Before publishing, every post must satisfy:

- [ ] Sounds like Nick talking to a friend (not AI-generated).
- [ ] Personal story in opening hook.
- [ ] Specific numbers, dates, and metrics included.
- [ ] Code examples tested and functional (if technical).
- [ ] Meta description under 160 characters.
- [ ] Proper heading hierarchy.
- [ ] Internal links to relevant content.
- [ ] Clear call to action or engagement question.
- [ ] Privacy guidelines respected (per `voice-and-style` policies).
- [ ] Mobile-friendly formatting.
