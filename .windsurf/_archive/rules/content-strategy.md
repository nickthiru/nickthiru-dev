# Content Strategy & Audience

## Target Audience

### Primary

- **Technical founders** building AI-powered products
- **Solo operators** running technical businesses
- **AI engineers** interested in production systems
- **Developers** transitioning to founder roles

### Characteristics

- Value authentic technical content
- Appreciate vulnerability and transparency
- Seek practical, actionable insights
- Have limited time and attention

## Content Tracks & Mix

- **60% Technical**: Deep dives, architecture, debugging, production patterns, code tutorials
- **30% Business**: Pricing, growth, building in public, solo founder operations, strategy
- **10% Personal**: Career transitions, imposter syndrome, work-life integration

## Platform Strategy

| Platform | Focus | Format |
|---|---|---|
| **Blog** | Long-form canonical content | 800-1500 word posts |
| **Newsletter** | Personal updates, exclusive content | Behind-the-scenes |
| **LinkedIn** | Professional insights | 150-300 word posts |
| **X/Twitter** | Quick insights, threads | 7-10 tweet threads |

## Content Pipeline

```
thiru-ai-labs/build-logs/ → Daily capture → Weekly synthesis → Blog post → Multi-platform repurpose
```

1. **Capture during development** — Zero-effort build log entries
2. **Daily transformation** — 15-30 min content creation (see `/capture-daily-content`)
3. **Weekly synthesis** — Build log review → canonical blog post (see `/plan-weekly-content`)
4. **Multi-platform distribution** — Repurpose for each platform

## Privacy & Sharing Guidelines

### Always Safe to Share

- Revenue/MRR (rounded: "$5K MRR" not "$5,234.12")
- User growth percentages and trends
- Technical process without proprietary details
- Mistakes made and what you learned
- Behind-the-scenes development journey

### Never Share Without Permission

- Customer names or identifying information
- Proprietary algorithms or technical IP
- Private user feedback or communications
- Legal, financial, or security-sensitive details

### Decision Framework

1. Could this help competitors? → Keep private
2. Could this create security risk? → Keep private
3. Could this identify specific customers? → Keep private
4. Is this about the pattern/learning? → Share the pattern
5. **Rule of thumb**: Share the pattern, not the exploitable detail
