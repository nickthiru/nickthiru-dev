# SEO & Quality Standards

## SEO Requirements (All Articles)

- **Meta description**: 150-160 characters, includes keywords naturally
- **Title tags**: 50-60 characters, compelling and keyword-rich
- **Headings**: Proper H1 → H2 → H3 hierarchy
- **Internal links**: Link to 2-3 relevant previous posts
- **Image optimization**: Alt text, descriptive filenames
- **URL structure**: Short, descriptive, includes keywords
- **Mobile-friendly**: Responsive formatting

## Voice + SEO Balance

- Keywords first, voice second — include keywords naturally in authentic voice
- Meta descriptions in my voice — write as if talking to a friend
- Headings as questions — conversational heading questions that include keywords
- Internal links with context — "Here's how I solved [keyword] problem"

## Technical Content Standards

- **Test all code examples** before publishing
- **Include imports** and necessary dependencies
- **Error handling**: Demonstrate proper error handling
- **Version info**: Specify relevant library/framework versions
- **Comments**: Add explanatory comments for complex code

## Content Type-Specific Standards

### Engineering Posts

- Problem → Failed Attempts → Breakthrough → Step-by-Step → Testing → Production → Lessons
- Show debugging process, not just final solution
- Explain WHY something works, not just THAT it works
- Connect technical topics to business benefits

### Business Posts

- Transparency Decision → What I Shared → Impact → Scary Part → Outcomes → Guidelines
- Always include specific metrics and data points
- Provide frameworks and mental models
- Share both successes and failures

### Product Posts

- What I Built → How It Works → Why It Matters → Challenge → Next Up
- Focus on user value and problem solving
- Show progress and momentum
- Include technical details without overwhelming

## Quality Checklist (All Posts)

### Before Publishing

- [ ] Sounds like me talking to a friend (not AI-generated)
- [ ] Personal story in opening hook
- [ ] Specific numbers, dates, and metrics included
- [ ] Code examples tested and functional (if technical)
- [ ] Meta description under 160 characters
- [ ] Proper heading hierarchy
- [ ] Internal links to relevant content
- [ ] Clear call to action or engagement question
- [ ] Privacy guidelines respected
- [ ] Mobile-friendly formatting
