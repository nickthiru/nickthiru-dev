## Content Standards

### Blog Post Structure

- **Hook**: Start with compelling question or problem
- **Context**: Why this topic matters to the audience
- **Solution**: Technical implementation with code examples
- **Results**: Real outcomes, metrics, and learnings
- **Takeaways**: Actionable insights for readers

### Writing Style Guidelines

- Write for technical founders and operators
- Use clear, concise language with technical depth
- Include practical examples and code snippets
- Share both successes and failures transparently
- Use storytelling to illustrate technical concepts
- Include actionable insights and takeaways

This file provides high-level voice philosophy and brand standards:

- **Authentic**: Write as a real person sharing real experiences
- **Technical Depth**: Provide substantive technical insights without oversimplifying
- **Vulnerable**: Share struggles, failures, and learning moments openly
- **Practical**: Focus on actionable insights and real-world applications
- **Conversational**: Write like you're talking to a peer over coffee

_Refer to the comprehensive style guide for detailed voice characteristics, signature phrases, "always do/never do" lists, and complete post structure templates._

### Language Patterns

- Use "I" statements for personal experiences
- Ask rhetorical questions to engage readers
- Use analogies from everyday life
- Include specific numbers and metrics when possible
- Balance technical terms with plain language explanations

### What to Avoid

- Corporate jargon and buzzwords
- Overly academic or formal language
- Pretending to have all the answers
- Hiding struggles or failures
- Writing as an authority rather than a peer
