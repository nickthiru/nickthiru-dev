# Procedure

## Overview

```text
Input: Published blog post from blog-writer
  ↓
Step 1: Read and extract    → Parse blog post for key insights, quotes, metrics
Step 2: LinkedIn adaptation → Create professional insight post (150–300 words)
Step 3: X/Twitter adaptation → Create thread (7–10 tweets) or single tweet
Step 4: Newsletter segment  → Write summary + queue for weekly newsletter
Step 5: Quality check       → Validate all versions against checklist
Step 6: Present for review  → User approves or requests changes
```

## LinkedIn Adaptation

**Duration:** 10 minutes.

### Step 1: Extract Core Insight

Identify the single most compelling insight from the blog post — the one thing a professional would stop scrolling for.

### Step 2: Write the Post

```
Opening hook (1–2 lines that stop the scroll)

2–3 short paragraphs with line breaks between them
- Use bullet points for clarity
- Keep paragraphs to 1–2 sentences

Key insight or lesson learned

Engagement question at the end?
```

### Step 3: Format

- Short paragraphs (1–2 sentences each)
- Line breaks between paragraphs for mobile readability
- 3–5 relevant hashtags at the end
- No external link in the body — add blog link as first comment after posting

### Templates

"I'm building" (re-activation):
```
I'm building [product] that solves [problem].

The challenge:
- [pain point 1]
- [pain point 2]

What I'm building instead:
- [solution approach]
- [key benefit]

If you're in [target audience]: what's your biggest blocker?
```

Build log progress (daily):
```
Build log: [Product Name]

Today I got [specific milestone] working.

What it does:
- [feature 1]
- [feature 2]

Why it matters: [user benefit]

Next up: [next milestone]

Question: would you rather [option A] or [option B]?
```

"I thought X, I was wrong" (learning):
```
I thought [assumption].

I was wrong.

What happened:
- [what you tried]
- [what broke]
- [what you changed]

Lesson: [one sentence key learning]

If you're building with [tech]: what's your hardest challenge right now?
```

## X/Twitter Adaptation

**Duration:** 10 minutes.

### Thread Structure (7–10 tweets)

```
Tweet 1: Hook + promise ("I spent 3 days debugging [X]. Here's what I learned 🧵")
Tweet 2–4: Key points from the blog post, one per tweet
Tweet 5–7: Lessons learned, specific and actionable
Tweet 8: Mistake or tradeoff that surprised you
Tweet 9: Summary or key takeaway
Tweet 10: CTA ("Full writeup on my blog: [link]")
```

### Single Tweet (for quick updates)

```
Build log: [Product Name]
Shipped [feature] today.
Next: [next milestone].
Biggest surprise: [key lesson].
```

### Rules

- Each tweet must stand alone and provide value independently.
- Use specific numbers and details — not vague claims.
- Include code snippets in tweets where relevant (use images for longer code).
- Thread hook must create curiosity or promise a specific benefit.

## Newsletter Compilation

**Duration:** 20 minutes. Performed weekly (Friday or Saturday).

### Structure

```markdown
# Weekly Newsletter

## Personal Note (2–3 sentences)
What you're working on, what's coming, or a behind-the-scenes moment
that doesn't appear on the blog.

## This Week's Posts

### [Post Title 1]
2-sentence summary of the key insight. What you'll learn and why it matters.

[Read the full post →](link)

### [Post Title 2] (if 2 posts that week)
2-sentence summary.

[Read the full post →](link)

## One Thing I Learned This Week
A single insight or lesson — the newsletter-exclusive content that gives
subscribers a reason to stay subscribed.

## Quick Links
- Follow on LinkedIn: [link]
- Follow on X: [link]
- Reply to this email with questions
```

### Rules

- Personal note is not from the blog — this is newsletter-exclusive.
- Blog summaries are 2 sentences maximum — enough to hook, not enough to replace the post.
- "One thing I learned" is a genuine insight, not marketing copy.
- Always end with "reply to this email" CTA — direct replies build the strongest relationships.

## Quality Checklist

Before presenting platform versions to the user:

- [ ] Each platform version stands alone (does not require the blog post to make sense)
- [ ] Voice matches `voice-and-style` policies across all versions
- [ ] LinkedIn post has no external link in the body
- [ ] X thread hook is compelling and specific
- [ ] Newsletter summary is concise (2 sentences per post)
- [ ] Newsletter has a personal note section
- [ ] Privacy guidelines respected across all versions
- [ ] No copy-paste between platforms — each is genuinely adapted
- [ ] Engagement element present (question or CTA) in LinkedIn and X versions
